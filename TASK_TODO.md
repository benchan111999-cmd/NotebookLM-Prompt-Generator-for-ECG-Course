## 🚀 智慧解析功能規劃 (PDF/Markdown 雙格式支援)

### 功能目標
實作課程大綱自動解析功能，支援 PDF 和 Markdown 兩種格式，自動提取模組和主題結構，減少手動輸入同時保持彈性和正確性。

### 技術實作計畫

#### 1. 檔案上傳元件 (`src/components/FileUploader.tsx`)
- 支援 .pdf 和 .md 檔案
- 檔案驗證 (類型、大小限制 - 25MB 上限)
- 拖放支援
- 載入狀態和錯誤處理

#### 2. 文字擷取服務
- **PDF 擷取**: `src/services/pdfTextExtractor.ts` (使用 pdfjs-dist + Web Worker)
- **Markdown 擷取**: `src/services/markdownTextExtractor.ts` (原生檔案讀取)

#### 3. 統一解析邏輯 (`src/shared/courseOutlineParser.ts`)
- 輸入: 擷取的純文字
- 輸出: 符合 `ModuleConfig` 格式的結構化資料
- 解析策略:
  * 模組標題識別 (Module X:, Chapter X:, 等)
  * 主題層次檢測 (縮排、編號、符號列表)
  * 信心度評分機制
  * 手動修正回退方案

#### 4. 解析結果確認UI (`src/components/ParsedDataReviewer.tsx`)
- 側邊欄比較: 原始文字 vs 解析結構
- 行內編輯模組/主題
- 拖放重新排序
- 接受/拒絕控制 + 會話持續性

#### 5. 資料流程整合
- 新增 React Context 管理解析後的課程資料
- 條件資料來源選擇 (解析資料 vs 硬編碼 courseData)
- 與現有模組/主題選擇器整合
- 保留手動選擇作為後備方案

#### 6. 提示詞生成適配
- 修改 `buildPrompt` 以接受動態課程資料
- 確保與既有 lecture/exercise 模式相容
- 維持所有現有功能

### 依賴與風險評估

#### 關鍵依賴
- `pdfjs-dist`: 前端 PDF 文字擷取 (約 2.2MB，考慮代碼分割)
- 無需額外依賴進行 Markdown 處理

#### 風險緩解
- **解析不準確**: 手動檢閱步驟確保正確性
- **PDF 變異性**: 回退到硬編碼資料保持現有功能
- **效能影響**: Web Worker 防止 UI 阻塞；懶載入優化初始載入
- **安全考量**: 客戶端處理使敏感課程大綱停留在用戶瀏覽器中

### 成功標準
1. 用戶可上傳 ECG 課程大綱 PDF/Markdown 檔案並提取結構化模組/主題資料
2. 提取資料從測試檔案中正確代表至少 80% 的模組/主題
3. 檢閱介面允許在 2 分鐘內修正解析錯誤
4. 使用解析資料生成的提示詞品質與手動選擇相當
5. 功能不會降低現有手動工作流程的效能
6. 所有現有功能保持完整且可測試

### 🔧 後端考量文件
為未來可能的服務器端 PDF 處理做準備：
- **安全考量**: 若移至後端處理，需確保文件上傳端點的安全（檔案類型驗證、大小限制、掃描惡意軟體）
- **效能考量**: 後端處理可減輕前端負擔，特別是處理大型 PDF 時，但需考慮延遲和頻寬使用

- **環境變數**: 後端可能需要額外的依賴（如 pdf-parse 花費較少的前端 bundle 大小）
